<h2>누구를 위해</h2>
아픈 가족을 돌보며 치료·행정·지원 정보를 스스로 찾아야 하는 보호자 및 환자.

<hr style="border: 0; height: 1px; background: #ccc; margin: 28px 0;">

<h2>누구의 어떤 문제 해결을 위해</h2>
<ul>
  <li>제도(산정특례·재난적의료비·지자체 지원)가 흩어져 있어 비교 어려움</li>
  <li>치료 단계가 바뀔 때마다 필요한 행정 절차가 달라지지만 보호자가 모두 기억해야 함</li>
  <li>보험 약관이 복잡하여 청구 가능성을 스스로 파악하기 어려움</li>
</ul>

<hr style="border: 0; height: 1px; background: #ccc; margin: 28px 0;">

<h2>어떤 기술을 사용해서</h2>
<ul>
  <li>React Native / Flutter</li>
  <li>Firebase(Firestore)</li>
  <li>Node.js(FaaS)</li>
  <li>AI 요약 기능</li>
</ul>

<hr style="border: 0; height: 1px; background: #ccc; margin: 28px 0;">

<h2>무엇 만들려고 하는가</h2>
<ol>
  <li>질병·치료 단계 선택에 따라 필요한 제도·행정 절차를 요약</li>
  <li>지원제도(산정특례·재난적의료비 등) 자동 매칭 안내</li>
  <li>간단한 보험 보장 체크 → 청구 후보 및 약관 확인 키워드 제공</li>
  <li>진료 일정, 산정특례 만료 알림 제공</li>
</ol>
